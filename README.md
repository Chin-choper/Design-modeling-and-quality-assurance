1. Головні файли 
manage.py Командний центр Django. Через цей файл виконуються всі команди: запуск сервера, створення міграцій, створення суперюзера та управління базою даних.

requirements.txt Список усіх бібліотек Python, необхідних для роботи проекту. Використовується для швидкого налаштування оточення командою pip install -r requirements.txt.

test_data Ключовий файл даних. Це текстовий файл, який містить дані про експорт та імпорт.

Принцип роботи: При завантаженні сторінки скрипт очищає базу даних і заново наповнює її даними з цього файлу. Це імітує роботу з динамічним зовнішнім джерелом.

uwsgi.ini Конфігураційний файл для сервера uWSGI. Вказує, як саме запускати проект, скільки процесів використовувати (4) та де знаходиться віртуальне оточення.

db.sqlite3 Локальна база даних. Тут зберігаються таблиці після застосування міграцій.

2. Логіка додатку, папка operations/
models.py Описує структуру бази даних. Тут створено клас Operation, який визначає поля:

op_type (Експорт/Імпорт/Завод);

category (Товар);

amount (Сума);

date (Дата). Також містить метод __str__ для зручного відображення в адмін-панелі.

views.py Мозок сайту.

Зчитує файл test_data і розбирає його.

Записує дані в БД.

Фільтрує дані за обраним роком та місяцем.

Розподіляє інформацію по вкладках і передає її в шаблон.

apps.py Стандартний файл конфігурації додатку operations.

3. Інтерфейс, папка templates/
operation_list.html Головний HTML-шаблон.

Використовує Bootstrap для дизайну таблиць та кнопкок.

Містить навігацію по вкладках Tabs.

Відображає фільтри за роками та місяцями.

Виводить таблиці з даними, які передав views.py.

4. Налаштування, папка djangosite/
settings.py Головний пульт керування проектом. Тут прописані:

Підключені додатки 'operations'.

Налаштування бази даних.

Шляхи до шаблонів.

Дозволені хости ALLOWED_HOSTS.

wsgi.py Точка входу для веб-сервера (uWSGI). Саме цей файл пов'язує код Python із веб-сервером.

urls.py Маршрутизатор. Вказує, що при заході на головну сторінку (/) треба викликати логіку з operations.views.

5. Системні файли Server Config
/etc/supervisor/conf.d/djangosite.conf Файл управління процесом. Дозволяє сайту працювати у фоновому режимі, автоматично запускатися після перезавантаження сервера та перезапускатися у разі помилок.


Для початку роботы потрібно встановити WSL Linux, та посля налаштучанная та завантаження проект с GIT встановити всі бібліотеки що є у файлі - requirements.txt

Робити це потрібно в дериткоріх самої программи, щоб потрапити туди source ~/lb1/bin/activate, cd ~/djangosite, після усіх встановленних біліотек сервер стратує командою - sudo supervisorctl start djangosite. 
Щоб дізнатися власний id введіть команду hostname -I, та підставте свої цифри в ось це посилання - http://000.000.000.000:9000/
